# בוט ושות' — העוזר המשפטי הדיגיטלי

## שם פנימי: בוט ושות'
## שם ללקוח: "העוזר המשפטי הדיגיטלי של משרד עו"ד גיא הרשקוביץ ושות'"
## סטטוס: תכנון (טרם נבנה)

## Overview
בוט WhatsApp שפונה ללקוח פוטנציאלי תוך 30 שניות מרגע כניסת הליד, אוסף פרטים, שולח תוכן ממוקד, ומתזכר לפני פגישות. כל המידע מתעדכן על אותו ליד ב-Firestore (לא יוצר כפילויות).

## הכלל המרכזי
```
הבוט לא מתאם — הבוט מעקב
✅ אוסף פרטים מהלקוח
✅ שולח תוכן (סרטונים, כתבות)
✅ מעדכן CRM
✅ מתזכר לפני פגישות
❌ לא מתאם פגישות
❌ לא דן במחירים
❌ לא מתערב בשיחת העובד עם הלקוח
```

## Unique Key = מספר טלפון
כל ליד מזוהה לפי טלפון. אם הטלפון כבר קיים ב-Firestore — מעדכן את אותו document, לא יוצר חדש.

## Full Flow — 6 שלבים

### שלב 1: שיחת חימום (בוט → לקוח, תוך 30 שניות)

**Trigger:** ליד חדש נשמר ב-Firestore עם טלפון + `botConversationDone: false`

```
בוט → לקוח:
"שלום! 👋
פנית למשרד עו"ד גיא הרשקוביץ ושות'.
נשמח לעזור לך! מה שמך המלא?"

לקוח: "מוראל דהאן"
→ עדכון Firestore: name = "מוראל דהאן"

בוט → לקוח:
"תודה מוראל!
באיזה נושא את פונה?
1️⃣ דיני עבודה
2️⃣ חוזים / מסחרי
3️⃣ מקרקעין / נדל"ן
4️⃣ הקמת עסק / שותפות
5️⃣ נושא אחר"

לקוח: "1" / "דיני עבודה" / "פוטרתי"
→ עדכון Firestore: category = "דיני עבודה"

בוט → לקוח (מותאם לנושא!):
"הבנתי, דיני עבודה.
תוכלי לתאר בקצרה מה קרה? (משפט אחד-שניים)"

לקוח: "פוטרתי בזמן הריון בלי שימוע"
→ עדכון Firestore: description = "..."

בוט → לקוח:
"קיבלתי, תודה מוראל! ✅
העברתי את הפרטים שלך לעו"ד מהצוות שלנו
שמתמחה בדיוק בנושא הזה.
יחזור/תחזור אליך בהקדם.
מתי נוח לך לשיחה?"

לקוח: "אחרי 14:00" / "בכל שעה"
→ עדכון Firestore: availableTime = "אחה"צ"
→ עדכון Firestore: botConversationDone = true
```

### שלב 2: שליחת תוכן ממוקד (מיד אחרי שלב 1)

```
בוט → לקוח:
"בינתיים, מוזמנת לראות:
🎥 עו"ד גיא הרשקוביץ — [כותרת סרטון לפי נושא]
[קישור YouTube]

📰 [כותרת כתבה לפי נושא]
[קישור כתבה]

נשמח לעזור! 😊
משרד עו"ד גיא הרשקוביץ ושות'"

→ עדכון Firestore: contentSent = ["video_id", "article_id"]
```

**מאגר תוכן לפי נושא:**
```
דיני עבודה:
  video: [קישור — טרם הוגדר]
  article: [קישור — טרם הוגדר]

מקרקעין:
  video: [קישור — טרם הוגדר]
  article: [קישור — טרם הוגדר]

מסחרי / חוזים:
  video: [קישור — טרם הוגדר]
  article: [קישור — טרם הוגדר]

שותפויות / חברות:
  video: [קישור — טרם הוגדר]
  article: [קישור — טרם הוגדר]

כללי:
  video: [קישור כללי על המשרד]
  article: [קישור כללי]
```

### שלב 3: עדכון CRM + קבוצת לידים

```
הודעה בקבוצת לידים:
"🔥 *ליד מוכן — מוראל דהאן*
📱 050-3420980
📌 דיני עבודה — פיטורין בהריון ללא שימוע
⏰ זמינה אחה"צ
⭐ 9/10 — ליד חם!
📋 מומלץ: חיים / אופק
🎬 צפתה בסרטון על זכויות עובדת בהריון"

DM לעובד מומלץ:
"📋 הכנה לשיחה — מוראל דהאן
📱 050-3420980
📌 דיני עבודה — פיטורין בהריון בלי שימוע
⏰ זמינה אחה"צ
🎬 צפתה בסרטון שלנו

💡 שאלות מומלצות:
• באיזה שבוע הריון?
• כמה זמן עובדת באותו מקום?
• יש מכתב פיטורין / תיעוד?
• האם היה שימוע?

💰 פגישת ייעוץ: 780₪ + מע"מ
⚖️ חוק עבודת נשים + חוק שוויון הזדמנויות"
```

### שלב 4: העובד מטפל (הבוט לא מתערב!)

```
חיים מעדכן בקבוצה         →  מה הבוט עושה
─────────────────────────────────────────────
"אני" / "מטפל"            →  CRM: assignedTo = חיים
                              DM הכנה לשיחה לחיים
                              לא פונה ללקוח!

"דיברתי, רוצה פגישה"      →  CRM: status = contacted
                              פולואפ לחיים מחר: "מה הסטטוס?"
                              לא פונה ללקוח!

"לא ענתה"                  →  CRM: status = no_answer
                              פולואפ לחיים אחרי 2 שעות
                              לא פונה ללקוח!

"ביקשה לחשוב"              →  CRM: status = followup
                              פולואפ לחיים אחרי 2 ימים
                              אפשרות: שליחת תוכן נוסף ללקוחה
                              (רק תוכן! לא לחץ)

"תואמה ושילמה"             →  CRM: status = closed
                              בוט שואל את חיים (לא הלקוחה!):
                              "מתי הפגישה? (תאריך + שעה)"
                              חיים: "5/4 ב-14:00"
                              CRM: meetingDate = 2026-04-05T14:00
                              → מפעיל תזכורות פגישה (שלב 5)

"לא רלוונטי"               →  CRM: status = not_relevant
                              לא פונה ללקוח!

"הפניה"                     →  CRM: status = closed, note = "הפניה חיצונית"
                              לא פונה ללקוח!
```

### שלב 5: תזכורות פגישה (בוט → לקוח)

**Trigger:** meetingDate קיים ב-Firestore

```
יום לפני הפגישה (09:00):
בוט → לקוח:
"שלום מוראל! 😊
תזכורת: מחר יש לך פגישה במשרד עו"ד גיא הרשקוביץ.
📅 5 באפריל, 14:00
📍 [כתובת המשרד]
📞 לשאלות: [טלפון המשרד]
מחכים לך!"

בוקר הפגישה (08:00):
בוט → לקוח:
"בוקר טוב מוראל!
היום פגישה ב-14:00.
📍 [כתובת המשרד]
🅿️ [הוראות חניה אם יש]
נתראה! 😊"

→ עדכון Firestore: meetingReminder1Sent = true, meetingReminder2Sent = true
```

### שלב 6: פולואפ ללקוח (מותנה בסטטוס)

**רק כשהעובד מעדכן סטטוס — הבוט מגיב ללקוח בהתאם:**

```
סטטוס: "ביקשה לחשוב" + עברו 2 ימים:
בוט → לקוח:
"היי מוראל! 😊
רצינו לבדוק — חשבת על הנושא?
אם יש שאלות אנחנו כאן.
🎥 בינתיים, עוד סרטון שיכול לעזור: [קישור]"

סטטוס: "לא ענתה" + עברו 4 שעות + העובד לא עדכן שוב:
בוט → לקוח:
"היי מוראל, ניסינו להשיג אותך.
מתי יהיה נוח לך לשיחה קצרה?"

סטטוס: "closed" (נסגרה עסקה):
לא שולחים כלום — העובד מטפל!
```

## Firestore Schema — שדות חדשים

```
leads/{docId}:
  ... (שדות קיימים) ...
  
  // שדות חדשים מבוט החימום:
  botConversationDone: boolean      // האם שיחת החימום הסתיימה
  botConversationStarted: boolean   // האם התחלנו שיחה
  description: string               // תיאור הבעיה (מהלקוח)
  availableTime: string             // מתי נוח לשיחה
  contentSent: [string]             // רשימת תכנים שנשלחו
  
  // שדות פגישה:
  meetingDate: timestamp            // תאריך ושעה
  meetingReminder1Sent: boolean     // תזכורת יום לפני
  meetingReminder2Sent: boolean     // תזכורת בוקר הפגישה
```

## מניעת כפילויות — Flow

```
ליד נכנס עם phone = "0503420980"
    ↓
חיפוש ב-Firestore: leads WHERE phone == "0503420980"
    ↓
├─ נמצא (docId: ABC123):
│   → עדכון ABC123 (לא יצירת ליד חדש)
│   → אם botConversationDone == false → שולח שיחת חימום
│   → אם botConversationDone == true → לא שולח שוב
│
└─ לא נמצא:
    → יצירת ליד חדש
    → שולח שיחת חימום
```

## מה לא לעשות (כללי בטיחות)

1. **לא לשלוח הודעה ללקוח אחרי 21:00 או לפני 08:00**
2. **לא לשלוח יותר מ-3 הודעות ללקוח ביום** (אנטי-ספאם)
3. **לא לשלוח תוכן אם הלקוח לא סיים את שיחת החימום**
4. **לא לפנות ללקוח אחרי שהעובד מעדכן "לא רלוונטי"**
5. **לא להתערב בשיחה כשהעובד מטפל** (בין "דיברתי" ל-"תואם")
6. **לא לשאול את הלקוח על תאריך/מחיר** — רק העובד עושה את זה
7. **אם הלקוח לא ענה על שיחת החימום — ניסיון אחד נוסף אחרי שעה, ואז עוצרים**

## שבת, חגים ושעות שקטות — כללים קריטיים!

**הבוט לא שולח הודעות ללקוחות ב:**
- שבת: משקיעת יום שישי עד מוצאי שבת (שעה אחרי השקיעה)
- חגים יהודיים: ראש השנה, יום כיפור, סוכות, פסח, שבועות, שמחת תורה
- ערבי חגים: אחרי 14:00
- יום שישי: אחרי 14:00
- כל יום: לפני 08:00 או אחרי 21:00

**לוגיקת שליחה:**
```
function canSendToClient() {
    // Israel timezone (Asia/Jerusalem)
    // Check: is it Shabbat? Holiday? Before 08:00? After 21:00? Friday after 14:00?
    // If any → return false, queue for next valid time
}
```

**תזכורת פגישה ליום ראשון:**
- לא שולחים בשבת!
- שולחים ביום **חמישי** (2 ימים לפני) — "ביום ראשון יש לך פגישה..."
- + בוקר ראשון 08:00

**הודעות שנחסמו בגלל שבת/חג → נכנסות לתור ונשלחות במוצ"ש/אחרי החג**

**חגים — רשימה (תאריכים עבריים, צריך ספריית hebcal או רשימה קבועה לשנה):**
```
2026:
- פסח: 1-8 אפריל (ערב: 31 מרץ אחה"צ)
- שבועות: 21-22 יוני
- ראש השנה: 12-13 ספטמבר
- יום כיפור: 21 ספטמבר
- סוכות: 26 ספטמבר - 3 אוקטובר
```
